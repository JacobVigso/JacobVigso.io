Creative Coding Experience Design IoT

IPASIF

Interactive Sound Installation Framework

The Challenge

Artists often struggle to create interactive sound installations due to the technical complexity of connecting sensors, audio synthesis, and visuals. We needed a modular toolkit that bridges the gap between physical performance and digital output.

The Solution

IPASIF is a wireless, modular framework that allows artists to plug in various sensors and immediately manipulate sound and visuals in real-time. We validated this by building three use cases, including "Ballin'"—a ball containing an accelerometer that modifies music based on how it is thrown.

Technical Stack

ESP32 Microcontrollers ESP-NOW Protocol PureData TouchDesigner OSC over UDP

Key Achievements

  • Collaborated weekly with professional artists (Lars Greve, Tina Tarpgaard) to iterate on the framework
  • Achieved average wireless latency of ~44ms for live performance responsiveness
  • Designed a "black box" sensor unit with intuitive X/Y/Z axis alignment
02
Virtual Reality Storytelling User Agency

TACET

VR Narrative Extension

The Challenge

How do you translate the passive viewing experience of a film into an active VR environment without breaking the narrative? The goal was to extend the universe of the short film TACET into a "Cinematic VR" experience.

The Solution

A VR experience where the user steps into the shoes of the protagonist, Adam. Unlike a film, the user navigates physical space to progress the story. The environment utilizes non-euclidean geometry (impossible spaces) to reflect the psychological state of the character.

Technical Stack

Meta Quest VR Unity Figma Spatial Audio Haptics

Design Philosophy

  • Worked directly with the film's director (Ahmed Kholy) to respect the original artistic vision
  • Created seamless transition from virtual cinema setting into the film's actual set
  • Focused on "Deep Co-Presence" using spatial audio and haptics for physical presence
03
Android Kotlin Mobile Dev

Paperback

On-Device Audiobook Player

The Challenge

Most audiobook apps demand an account, cloud sync, or lock you into a single store. Listeners who already own their .m4b/.mp3 libraries just want a clean, private player that works fully offline—with proper chapter support and the system media controls a real audio app should have.

The Solution

A 100% on-device audiobook player built in Kotlin and Jetpack Compose. Media3/ExoPlayer drives playback through a MediaSessionService, so lock-screen, notification, Bluetooth and Android Auto controls all work out of the box. A hand-written MP4 "box" parser reads chapters straight from the file—no server, no companion app, no cloud sync.

Technical Stack

Kotlin Jetpack Compose Media3 / ExoPlayer Room MediaSessionService App Widgets

Key Achievements

  • Wrote a custom MP4 chapter parser from scratch (Nero chpl + QuickTime chap fallback) that reads only box headers—never the audio itself
  • Full background playback with system media controls, plus two home-screen widgets (Now Playing + scrollable Library)
  • Sleep timer (fixed or end-of-chapter), variable speed (0.5–3×), auto-rewind on resume, and per-book progress persisted locally with Room
04
AI / LLM Full-Stack Web Scraping

Tilbud Finder

AI Meal Plans from Weekly Grocery Offers

View Live Site

The Challenge

The cheapest way to shop is to cook around whatever is on sale in the weekly tilbudsavis. But cross-referencing offers across several Danish supermarket chains and turning them into actual meals is tedious, manual work most people never bother with.

The Solution

A full-stack web app that scrapes current offers from Danish chains (Føtex, Netto, MENY, Bilka) with Puppeteer, filters them down to real food items, then feeds the discounted ingredients to a locally-run LLM (Ollama / Llama 3.2) that generates budget-friendly recipes. The scrape-and-generate pipeline runs locally with no paid APIs; the finished recipes ship as a fast static site.

Technical Stack

Node.js Express Puppeteer Ollama (Llama 3.2) Vanilla JS Netlify

Key Achievements

  • Built an automated scraping pipeline pulling live offers from four supermarket chains via eTilbudsavis
  • Free, self-hosted AI: recipes are generated locally with Ollama (Llama 3.2)—no paid API keys—then published as static JSON
  • Danish-language UI with per-store offer previews and one-click recipe generation
